# Repositorios interesantes de SAP Business One

Recopilación de repositorios, librerías, frameworks y herramientas relacionadas con SAP Business One.

## Service Layer

### [B1 API Cookbook](https://github.com/Xaypanya/b1-api-cookbook)

Recurso completo para desarrolladores que trabajan con las APIs de SAP Business One Service Layer.

Incluye ejemplos, colecciones de Postman y casos prácticos de integración.

### [B1SLayer](https://github.com/bgmulinari/B1SLayer)

Cliente ligero para SAP Business One Service Layer desarrollado en .NET.

### [B1 Service Layer](https://github.com/nestebe/b1-service-layer)

Librería JavaScript para trabajar con la API de SAP Business One Service Layer.

### [SAP Business One Service Layer Proxy](https://github.com/gregorwolf/sap-business-one-service-layer-proxy)

Proxy inverso para SAP Business One Service Layer.

Puede resultar útil para gestionar aspectos como:

* CORS.
* Certificados.
* Autenticación.
* Exposición controlada de Service Layer.
* Enrutamiento de peticiones.

### [B1 Service FLayer](https://github.com/nemethmik/b1serviceflayer)

Librería para Dart que facilita el desarrollo contra SAP Business One Service Layer.

## DI API y desarrollo de add-ons

### [SAP Business One DI API Helper](https://github.com/opxw/sapb1-diapi-helper)

Librería auxiliar para simplificar el desarrollo con SAP Business One DI API.

### [B1Base](https://github.com/WilliamWachholz/B1Base)

Framework moderno en C# para desarrollar add-ons de SAP Business One mediante UI API y DI API.

Es especialmente interesante para estudiar:

* Abstracción de formularios y eventos.
* Inyección de dependencias.
* Separación entre infraestructura y lógica de negocio.
* Organización de proyectos SAP Business One.
* Estructuración de add-ons mantenibles.
* Gestión de servicios y dependencias.

Es una alternativa moderna frente a algunos frameworks históricos de desarrollo para SAP Business One.

### [Dover](https://github.com/efpiva/dover)

Framework de desarrollo para SAP Business One.

Puede utilizarse como referencia para estudiar la arquitectura y organización de add-ons desarrollados con UI API y DI API.

## Inteligencia artificial y automatización

### [SAP Business One Sales AI Automation](https://github.com/atonekaboni/SBO_Sales_AI_Automation)

Proyecto de automatización de ventas para SAP Business One mediante inteligencia artificial.

### [B1 Assistant Lite](https://github.com/SAP-archive/b1-assistant-lite)

Asistente ligero para SAP Business One desarrollado originalmente por SAP.

El repositorio está archivado, pero puede resultar útil como referencia para estudiar asistentes y automatizaciones aplicadas a SAP Business One.

## Integraciones

### [SAP Woo](https://github.com/replantadev/sapwoo)

Conector profesional entre WooCommerce y SAP Business One.

Puede resultar interesante para analizar:

* Sincronización de productos.
* Sincronización de clientes.
* Pedidos de venta.
* Stock.
* Precios.
* Integraciones entre comercio electrónico y SAP Business One.

## Crystal Reports

### [Crystal Reports Ninja](https://github.com/rainforestnet/CrystalReportsNinja)

Aplicación de consola de código abierto para Windows, desarrollada con .NET Framework 4.5.

Permite cargar informes externos de Crystal Reports en formato `.rpt` y realizar operaciones como:

* Exportar informes a PDF.
* Exportar informes a Excel.
* Exportar a otros formatos.
* Imprimir directamente en una impresora.
* Automatizar la generación de informes desde línea de comandos.

Aunque no es una herramienta exclusiva de SAP Business One, puede ser útil para automatizar informes de Crystal Reports utilizados dentro de SAP B1.

## Instalación y despliegue

### [SAP Business One for SAP HANA Chef Cookbook](https://github.com/SAP-archive/business-one-hana-chef-cookbook)

Cookbook de Chef para automatizar la instalación de SAP Business One para SAP HANA.

El repositorio está archivado, pero puede resultar útil como referencia histórica para estudiar procesos automatizados de instalación.

### [SAP Business One Chef Cookbook](https://github.com/SAP-archive/business-one-chef-cookbook)

Cookbook de Chef para automatizar la instalación de SAP Business One.

Al igual que el anterior, se trata de un repositorio archivado orientado principalmente a despliegues históricos de SAP Business One.

## Leyenda

* **Service Layer:** clientes, ejemplos y herramientas para consumir la API REST/OData de SAP Business One.
* **DI API:** librerías y utilidades para trabajar con la API COM de SAP Business One.
* **UI API:** herramientas para desarrollar formularios, menús y eventos dentro del cliente de SAP Business One.
* **Integraciones:** conectores con plataformas externas.
* **Archivado:** repositorios sin mantenimiento activo que siguen siendo útiles como referencia técnica.
